# Optional Chaining

Es una característica que permite acceder a propiedades de un objeto o invocar funciones sin causar errores si alguna de las referencias en la cadena es null o undefined.

Este operador se representa con el signo '?'. Con él, puedes acceder de manera segura a propiedades o métodos de un objeto sin tener que comprobar manualmente si cada nivel en la cadena es válido.

```javascript
const usuario = {
  nombre: 'Juan',
  direccion: {
    ciudad: 'Madrid'
  }
};

console.log(usuario.direccion?.ciudad);  // "Madrid"
console.log(usuario.telefono?.numero);   // undefined

```

1. usuario.direccion?.ciudad devuelve 'Madrid', porque direccion existe.
2. usuario.telefono?.numero devuelve undefined sin arrojar un error, ya que telefono no existe en el objeto.

El encadenamiento opcional se puede usar con objetos, arrays y funciones, y previene errores cuando se trata de valores que podrían no existir. Es muy útil cuando trabajas con datos que pueden tener estructuras dinámicas o incompletas.

```javascript

const objeto = {
  saludar: () => 'Hola',
};

console.log(objeto.saludar?.());
// "Hola"

console.log(objeto.adios?.());
// undefined

```

En este caso, si adios no existe, no se lanza un error, sino que simplemente devuelve undefined.

## Métodos

Podemos verificar con el '?.' para ver si un método existe en un objeto y lo combinamos con el '??' (Nullish Coalesign) para asignar un mensaje por defecto.


En este caso no obtenemos Undefined como en el ejemplo inicial.

```javascript

const objeto = {
    saludar: () => 'Hola',
  };
  
  console.log(objeto.saludar?.());
  // "Hola"
  
  console.log(objeto.adios?.() ?? 'No existe el método adios');

```


# Arrays

También podemos usar la misma lógica con los arreglos, para ver si existe un valor dentro del array, regresar ese valor, y sino se usa el Nullish Coalesign para regresar un valor por defecto.

```javascript

const usuario = [ {nombre: 'Pedro', edad: 25,}]

console.log(usuario[0]?.nombre ?? 'No hay nombre')

console.log(usuario[0]?.correo ?? 'No hay correo')

/*

Resultado:

'Pedro'
'No hay correo'

*/

```
