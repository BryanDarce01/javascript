# Lo que pasa al crear una clase

## Proceso al crear una clase o usar `new`:

```javascript

class Persona {
  constructor(nombre) {
    this.nombre = nombre;
  }

  saludar() {
    console.log(`Hola, soy ${this.nombre}`);
  }
}

const p1 = new Persona("Bryan");

```

Cuando se ejecuta `new Persona("Bryan")`, pasan estas cosas:

1. **Se crea un nuevo objeto vacío**
    
    Internamente, JS hace algo como:

```javascript

    let obj = {};

```

2. **Ese objeto queda enlazado al prototipo de la clase**

    JS hace:

```javascript

Object.setPrototypeOf(obj, Persona.prototype);

```

→ Esto asegura que `p1` tenga acceso a los métodos definidos en `Persona.prototype`, como `saludar()`.

3. **Se llama al constructor de la clase con `this` apuntando al nuevo objeto**

```javascript

Persona.call(obj, "Bryan");

```

4. **El objeto ya inicializado se devuelve automáticamente**

→ No se necesita usar `return` (a menos que se devuelva un objeto diferente manualmente)

## 👉 En resumen:

Cuando se usa `new`, **JS** hace toda la “magia” de instanciar, enlazar al prototipo y devolver el objeto nuevo.
