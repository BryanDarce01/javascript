# Statis Methods

En JavaScript, cuando hablamos de propiedades o métodos estáticos (static) en una clase u objeto, nos referimos a aquellos que pertenecen directamente a la clase (o al objeto), no a las instancias creadas a partir de ella.

## Concepto:

- **Métodos normales de clase**: se heredan en el prototipo y los puede usar cada objeto creado con `new`.

- **Métodos estáticos**: se asocian solo a la clase, no a los objetos creados.

```javascript

class Person {
  constructor(name) {
    this.name = name;
  }

  // Método de instancia (cada objeto lo puede usar)
  sayHello() {
    console.log(`Hi, I'm ${this.name}`);
  }

  // Método estático (solo existe en la clase Person, no en los objetos)
  static species() {
    return "Homo sapiens";
  }
}

const pedro = new Person("Pedro");

pedro.sayHello(); // ✅ "Hi, I'm Pedro"
console.log(Person.species()); // ✅ "Homo sapiens"
console.log(pedro.species);    // ❌ Error (no existe en la instancia)

```

Aquí `species` no está disponible en `pedro`, solo en la clase `Person`.

## Ejemplo con objeto estático (sin clases):

En objetos también se puede definir directamente:

```javascript

const MathUtils = {
  add(a, b) {
    return a + b;
  },
  multiply(a, b) {
    return a * b;
  }
};

console.log(MathUtils.add(2, 3));      // 5
console.log(MathUtils.multiply(2, 3)); // 6

```

Aquí `MathUtils` funciona como un objeto estático, porque no necesitamos instanciarlo con `new`.

👉 Esto es lo mismo que pasa con objetos nativos como `Math` o `JSON`.


## Resumen:

- **Getters/Setters** → controlan acceso a propiedades.

- **Métodos normales** → viven en el prototype de las instancias.

- **Métodos estáticos** → viven en la clase misma, no en las instancias.

Ejemplos nativos de objetos estáticos: `Math`, `Date.now()`, `JSON`.
