# Programación Orientada a Objetos (POO u OOP, Object-Oriented Programming)

Es un **paradigma de programació**n que organiza el código alrededor de objetos, en lugar de funciones o lógica suelta.

Un objeto es una entidad que combina:

- **Atributos** → Sus datos o propiedades.

- **Métodos** → Sus funciones o comportamientos.

![oop](/img/oop.png)

## 📌 Principios fundamentales de la POO

## Abstracción

Simplificar la complejidad mostrando solo lo importante.

Ejemplo: para conducir no necesitas saber cómo funciona cada pistón del motor.

![abstraccion](/img/abstraccion.png)

## Encapsulación

Ocultar los detalles internos de un objeto y exponer solo lo necesario.

Ejemplo: un coche tiene un motor, pero solo usas el volante y pedales.


![encapsulacion](/img/encapsulacion.png)

## Herencia

Permite crear nuevas clases a partir de otras existentes.

Ejemplo: una clase Animal puede ser base para Perro y Gato.


![herencia](/img/herencia.png)

## Polimorfismo

Diferentes clases pueden usar un mismo método, pero con comportamientos distintos.

Ejemplo: animal.hacerSonido() puede ser ladrar() en Perro y maullar() en Gato.


![polimorfismo](/img/polimorfismo.png)

## 📖 Ejemplo

```javascript

// Clase base
class Animal {
  constructor(nombre) {
    this.nombre = nombre;
  }

  hacerSonido() {
    console.log("Este animal hace un sonido.");
  }
}

// Herencia
class Perro extends Animal {
  hacerSonido() {
    console.log("Guau 🐶");
  }
}

class Gato extends Animal {
  hacerSonido() {
    console.log("Miau 🐱");
  }
}

// Uso
const perro = new Perro("Firulais");
const gato = new Gato("Mishito");

perro.hacerSonido(); // Guau 🐶
gato.hacerSonido();  // Miau 🐱

```

## 📌 Ventajas de la POO

- ✅ Código más organizado.
- ✅ Reutilización gracias a la herencia.
- ✅ Facilidad de mantenimiento y escalabilidad.
- ✅ Mejor modelado del mundo real.
