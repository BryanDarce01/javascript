# Object.create()

`Object.create()` en JavaScript es un método estático que se usa para crear un nuevo objeto, estableciendo explícitamente el prototipo de ese objeto.

## 📌 Sintaxis:

```javascript

Object.create(proto, propertiesObject)

```

- `proto`: El objeto que se usará como prototipo del nuevo objeto.

- `propertiesObject` (opcional): Define nuevas propiedades para el objeto recién creado (similar a `Object.defineProperties`).

## Ejemplo básico:

```javascript

const personPrototype = {
  greet() {
    console.log(`Hi, my name is ${this.name}`);
  }
};

const person = Object.create(personPrototype);
person.name = "Bryan";
person.greet(); // Hi, my name is Bryan

```

### 👉 Aquí:

- `personPrototype` es el prototipo.

- `person` se crea con ese prototipo.

- Cuando llamamos `person.greet()`, JS no encuentra greet en person, pero sigue la **prototype chain** hasta `personPrototype`

## Con propertiesObject:

```javascript

const car = Object.create(Object.prototype, {
  make: {
    value: "Toyota",
    writable: true,
    enumerable: true,
    configurable: true
  },
  year: {
    value: 2022,
    writable: false
  }
});

console.log(car.make); // Toyota
console.log(car.year); // 2022

```

## Diferencia vs `{}` literal:

```javascript

const obj1 = {};  
console.log(Object.getPrototypeOf(obj1) === Object.prototype); // true

const obj2 = Object.create(null); 
console.log(Object.getPrototypeOf(obj2)); // null

```

- `Object.create(null)` crea un objeto sin prototipo, útil cuando quieres un objeto puro (sin heredar métodos como toString, hasOwnProperty, etc.).

## En resumen:

- `Object.create()` te da control total sobre el prototipo de un objeto.

- Es muy usado para implementar herencia prototípica sin necesidad de clases o constructores.
