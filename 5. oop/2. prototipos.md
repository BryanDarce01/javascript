# Herencia de Prototipos

En ***JavaScript***, ahora tenemos la sintaxis de `class`, JavaScript no usa clases reales como en Java o C#.

En realidad, todo en JS se basa en la herencia de prototipos.

## 📌 ¿Qué es la herencia de prototipos?

Es el **mecanismo interno de JavaScript** por el cual los objetos pueden heredar propiedades y métodos de otros objetos.

Cada **objeto** en **JS** tiene una referencia interna llamada `[[Prototype]]` (a veces accesible con ` __proto__`), que apunta a otro objeto.

Si buscas una propiedad en un objeto y no está allí, JS la busca en su prototipo, y así sucesivamente hasta llegar al prototipo base (Object.prototype).

![prototipos](/img/prototipos.png)

### 👉 Esto crea la cadena de prototipos (prototype chain).

## 📖 Ejemplo básico

```javascript

// Objeto padre
const animal = {
  sonido: function () {
    console.log("El animal hace un sonido");
  }
};

// Crear un nuevo objeto que herede de animal
const perro = Object.create(animal);
perro.ladrar = function () {
  console.log("Guau 🐶");
};

perro.sonido(); // Heredado -> "El animal hace un sonido"
perro.ladrar(); // Propio -> "Guau 🐶"

```

### Aquí:

- `perro` no tiene el método `sonido`, pero lo hereda de `animal` gracias al prototipo.

## 📌 Resumen corto

- La herencia de prototipos es la forma en que JS organiza la herencia.

- Cada objeto tiene un prototipo, que es otro objeto.

- Si no encuentra una propiedad en el objeto, la busca en su prototipo (y así hasta `Object.prototype)`.

- `class` es solo azúcar sintáctica sobre este mecanismo.

# IMPORTANTE

**El corazón de cómo funciona la herencia prototípica en JavaScript.**

**LA PROPIEDAD `prototype` DE UN CONTRUCTOR, ES LA MISMA QUE `__proto__` DE LOS OBJETOS CREADOS A PARTIR DEL CONSTRUCTOR**

```javascript

console.log(arr.__proto__ === Array.prototype); // true

```

## 1. `prototype` en un constructor

- En funciones constructoras (incluyendo clases en ES6), la propiedad especial .prototype es un objeto que contiene métodos y propiedades que serán compartidos por todas las instancias creadas con new.

- Ejemplo:

```javascript

function Persona(nombre) {
  this.nombre = nombre;
}

Persona.prototype.saludar = function () {
  console.log("Hola, soy " + this.nombre);
};


```

- Aquí, `Persona.prototype` contiene el método saludar.

## 2. **`__proto__` en objetos**

- Cada objeto en JS tiene una propiedad interna oculta llamada `[[Prototype]]`, a la cual podemos acceder con `__proto__ `(no recomendado en producción, pero útil para aprender).

- Esta propiedad apunta al objeto prototipo del constructor que creó ese objeto.

```javascript

const juan = new Persona("Juan");

console.log(juan.__proto__ === Persona.prototype); // ✅ true

```

## 3. **Cadena de prototipos**

Si intentas acceder a una propiedad que no existe en el objeto, JS la busca en su `__proto__`.

Si no está ahí, sigue subiendo en la cadena `(__proto__.__proto__)` hasta llegar a `Object.prototype`.

Ejemplo:

```javascript

const arr = [1, 2, 3];

console.log(arr.__proto__ === Array.prototype); // ✅ true
console.log(arr.__proto__.__proto__ === Object.prototype); // ✅ true
console.log(Object.prototype.__proto__); // null -> fin de la cadena

```

## 📌 Conclusión:

- `.prototype` vive en las funciones constructoras.

- `__proto__ `vive en las instancias creadas con esas funciones.

### Y sí: el `prototype` **del constructor ===** `__proto__` de las instancias 🚀.
