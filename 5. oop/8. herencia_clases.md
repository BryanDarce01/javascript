# Herencia de Clases

Este es un ejemplo **clásico de cómo implementar herencia de clases en JavaScript usando funciones constructoras y prototipos**, antes de que existiera la sintaxis `class`.


## 1. Definición del "padre" → Person

Aquí se define un constructor que crea objetos `Person` con propiedades `name` y `age`.

```javascript

const Përson = function(name, age) {
    this.name = name;
    this.age = age;
};

```

## 2. Método en el prototipo de Person

``` javascript

Person.prototype.calcAge = function() {
    console.log(`My age is ${2024 - this.age}`);
};

```

- Esto hace que todas las instancias de `Person` compartan el mismo método `calcAge` gracias a la herencia prototípica.

- Si creas 1000 personas, no se duplicará el método en memoria, todas apuntan al mismo.


## 3. Definición del "hijo" → Student

``` javascript

const Student = function(name, age, course) {
    Person.call(this, name, age);
    this.course = course;
}

```

- Aquí se usa `Person.call(this, name, age)` para heredar las propiedades de `Person` dentro de `Student`.

- Sin esto, `Student` no tendría `name` ni `age`.


## 4. Conexión de prototipos (herencia de métodos)

``` javascript

Student.prototype = Object.create(Person.prototype);

```

- Esto es la clave de la herencia en prototipos.

- Con `Object.create(Person.prototype)` decimos:
- 👉 El prototipo de `Student` es un objeto que tiene como prototipo a `Person.prototype.`

Así, cualquier instancia de `Student` puede usar métodos definidos en `Person.prototype`.

## 5. Método propio de Student

``` javascript

Student.prototype.introduce = function() {
    console.log(`My name is ${this.name} and I study ${this.course}`);
}

```
- Esto solo existe en `Student.prototype`, no en `Person.prototype`.

## 6. Prueba de instancias

``` javascript

const mike = new Student('Mike', 1995, 'Computer Science');
mike.introduce(); // My name is Mike and I study Computer Science
mike.calcAge(); // My age is 29

```
- `introduce()` viene de `Student.prototype`.

- `calcAge()` viene de `Person.prototype`.

## 7. Comprobaciones de herencia

``` javascript

console.log(mike instanceof Student); // true
console.log(mike instanceof Person); // true

```

- ✅ `mike` es instancia de ambos porque `Student` hereda de `Person`.


## 8. Cadena de prototipos

``` javascript

console.log(mike.__proto__); // Student.prototype
console.log(mike.__proto__.__proto__); // Person.prototype

```

### La cadena es así:

``` javascript

mike --> Student.prototype --> Person.prototype --> Object.prototype --> null

```

## 📌 En resumen:

- Propiedades se heredan con `Person.call(this, ...)`.

- Métodos se heredan conectando prototipos con `Object.create(Person.prototype)`.

- Esto es la base de la herencia clásica en JS antes de `class`.
