# Top-Level Await (TLA)

Normalmente, en JavaScript no se puede usar `await` fuera de una función `async`, por ejemplo:

``` javascript

// ❌ Esto da error si no estás en un módulo
const data = await fetch("https://api.example.com/data");

```
- Pero desde ES2022, en los módulos ES (`.mjs` o con `"type": "module"`),

- ¡ahora sí se puede usar `await` **directamente** al nivel superior del archivo!

## Ejemplo simple

``` javascript

// ejemplo.js
const respuesta = await fetch("https://restcountries.com/v3.1/name/guatemala");
const datos = await respuesta.json();
console.log(datos[0].name.common); // "Guatemala"

```

- Aquí `await` se ejecuta fuera de cualquier función, directamente en el nivel superior del módulo.

- No hace falta `async function`.

# ¿Por qué se usa?

El **Top-Level Await** simplifica el trabajo con operaciones asíncronas
como `fetch`, `import()`, o lectura de archivos, sin tener que envolver todo en `async` funciones.

## Antes (sin TLA)

``` javascript

(async function() {
  const respuesta = await fetch("https://api.com/data");
  const data = await respuesta.json();
  console.log(data);
})();

```

## Ahora (con TLA)

``` javascript

const respuesta = await fetch("https://api.com/data");
const data = await respuesta.json();
console.log(data);

```

# 🔄 Encadenamiento entre módulos

Otra característica poderosa:

- Si un módulo usa `await` al nivel superior, **otros módulos que lo importen deberán esperar a que termine**.

# Ejemplo

``` javascript

// data.js
console.log("Iniciando fetch...");
const res = await fetch("https://restcountries.com/v3.1/name/mexico");
const data = await res.json();
console.log("Datos listos");
export default data[0].name.common;

```

``` javascript

// app.js
import pais from "./data.js";

console.log("Mostrando país:", pais);

```

## 🧩 Resultado en consola

``` javascript

Iniciando fetch...
Datos listos
Mostrando país: Mexico

```

- En este caso `app.js` espera a que `data.js` termine su `await` antes de ejecutarse.

# ⚙️ Dónde funciona

## ✅ Funciona en:

- Archivos `.mjs`

- Archivos con `"type": "module"` en el package.json

- En navegadores modernos (en `<script type="module">`)

## ❌ No funciona en:

- Archivos comunes `.js` en entorno CommonJS (Node sin `"type": "module"`)

- Bloques de código normales sin módulo

# ⚠️ Precauciones

- Si se usa TLA, el módulo que lo contiene se vuelve asíncrono,
y otros módulos deben esperar su resolución antes de continuar.

- Puede afectar el rendimiento de carga si no se usa con cuidado.

# 💡 En resumen

| Concepto       | Explicación                                                          |
| -------------- | -------------------------------------------------------------------- |
| **Qué es**     | Permite usar `await` directamente en el nivel superior del módulo    |
| **Dónde**      | Solo en módulos ES (`type: module` o `.mjs`)                         |
| **Ventaja**    | Simplifica el código asíncrono (sin `async` envolvente)              |
| **Efecto**     | Hace que la carga del módulo sea asíncrona                           |
| **Uso típico** | `fetch()`, `import()`, inicializaciones que requieren datos externos |
