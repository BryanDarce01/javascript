# JSDoc

**JavaScript Documentation** es una herramienta y una sintaxis de comentarios que te permite documentar código **JavaScript** de forma clara, estructurada y legible tanto para humanos como para herramientas automáticas (como **VS Code**, **TypeScript** o generadores de documentación).

## 🧠 En pocas palabras

**JSDoc** sirve para describir qué hace el código: funciones, clases, parámetros, tipos de datos, valores de retorno, etc.

## 🧩 Ejemplo:

``` javascript

/**
 * Suma dos números.
 * @param {number} a - Primer número.
 * @param {number} b - Segundo número.
 * @returns {number} La suma de ambos números.
 */

function sumar(a, b) {
  return a + b;
}

console.log(sumar(5, 10)); // 15

```

🧠 Aquí:

- `/** ... */` → Indica un bloque de documentación JSDoc.

- `@param` → Describe cada parámetro de la función.

- `@returns` → Indica el tipo y significado del valor devuelto.

**VS Code** o **WebStorm**, por ejemplo, mostrarán esa descripción al pasar el cursor sobre `sumar()`.

## 🧰 Comandos más usados de JSDoc

| Etiqueta                            | Significado                           |
| ----------------------------------- | ------------------------------------- |
| `@param {Tipo} nombre`              | Parámetro de la función               |
| `@returns {Tipo}`                   | Valor que devuelve                    |
| `@type {Tipo}`                      | Define el tipo de una variable        |
| `@property {Tipo} nombre`           | Define propiedades de un objeto       |
| `@deprecated`                       | Marca algo como obsoleto              |
| `@example`                          | Muestra un ejemplo de uso             |
| `@async`                            | Indica que la función es asincrónica  |
| `@throws`                           | Describe los errores que puede lanzar |
| `@private`, `@public`, `@protected` | Control de visibilidad (en clases)    |


## ⚙️ Generar documentación automáticamente

### Si se instala JSDoc globalmente:

`npm install -g jsdoc`


Y luego ejecutar:

`jsdoc archivo.js -d docs`

- Esto generará una carpeta `/docs` con documentación HTML profesional basada en los comentarios.
