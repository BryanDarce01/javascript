# CLOSURES

Un **closure** es una función que recuerda el ámbito (scope) en el que fue creada, aunque se ejecute fuera de ese ámbito.

Dicho de forma más práctica:

- Un **closure** "encierra" variables del entorno en el que fue definido, lo que permite acceder a esas variables incluso después de que ese entorno haya terminado su ejecución.

# 🔍 ¿Por qué es importante esto?

## Porque en JavaScript las funciones pueden:

- Ser retornadas por otras funciones.

- Ser asignadas a variables.

- Ser ejecutadas después de que su función padre haya terminado.

- Y cuando eso sucede, aún recuerdan las variables del lugar donde fueron creadas. Eso es un closure.

``` javascript

function saludar(nombre) {
  return function() {
    console.log("Hola " + nombre);
  };
}

const saludoBryan = saludar("Bryan");
saludoBryan(); // "Hola Bryan"

```

¿Qué pasó aquí?
1. `saludar("Bryan")` ejecuta la función y retorna una nueva función anónima.

2. Esa función anónima recuerda el valor de nombre, aunque la función saludar ya terminó de ejecutarse.

3. Ese "recuerdo" de la variable nombre es el **closure**.


## Otro ejemplo

``` javascript

function crearContador() {
  let contador = 0;

  return function() {
    contador++;
    console.log(contador);
  };
}

const contar = crearContador();

contar(); // 1
contar(); // 2
contar(); // 3

```

¿Qué sucede?
1. `crearContador()` devuelve una función que mantiene el acceso a la `contador`.

2. Aunque `crearContador()` terminó, la variable `contador` vive dentro del closure.



# 🧰 ¿Para qué sirven los **closures**?

- Encapsular estado (como un contador).

- Proteger variables privadas (no accesibles desde fuera).

- Crear funciones personalizadas con datos precargados.

- Funciones en bucles (evitar errores comunes con **var**).


# ⚠️ Precauciones

- Los **closures** pueden causar fugas de memoria si mantienen referencias innecesarias a objetos grandes.

- Usar **closures** de forma incorrecta en bucles puede dar resultados inesperados (por eso se usa `let` en vez de `var`).


# 📌 En resumen

- **Qué es:**  Función que recuerda el contexto en el que fue creada.

- **Cuándo se forma:** Al retornar una función desde otra o usar funciones anidadas.


- **Qué permite:** Acceder a variables "externas" incluso si la función externa ya terminó.

- **Uso común:** Contadores, funciones privadas, manejadores de eventos.
