# First Class Functions

Las funciones de primera clase son simplemente una característica que un lenguaje de programación puede tener o no tener.
Lo único que significa es que todas las funciones son valores.
En la práctica, no existen "funciones de primera clase" como tal; es solo un concepto.

# Higher Order Functions

Las funciones de orden superior son posibles en la práctica porque el lenguaje admite funciones de primera clase.

## ¿Qué significa First Class Functions (Funciones de primera clase)?

Cuando decimos que las funciones son de primera clase, estamos diciendo que las funciones se pueden tratar como cualquier otro valor en el lenguaje. Es decir, en JavaScript puedes:

1. Guardar una función en una variable
2. Pasar una función como argumento a otra función
3. Devolver una función desde otra función

```javascript

// Guardamos una función en una variable
const saludar = function() {
  console.log("¡Hola!");
};

// La ejecutamos
saludar(); // ¡Hola!

```

Aquí __saludar__ es una variable que contiene una función. Eso es posible porque en JavaScript las funciones son valores, igual que los números o las cadenas.


## ¿Qué son Higher Order Functions (Funciones de orden superior)?
Las funciones de orden superior son funciones que reciben otras funciones como argumentos, o devuelven funciones.

➡️ Esto es posible gracias a que las funciones son de primera clase.

➡️ Ejemplo de función de orden superior:

```javascript

function operar(num1, num2, operacion) {
  return operacion(num1, num2);
}

function sumar(a, b) {
  return a + b;
}

function restar(a, b) {
  return a - b;
}

console.log(operar(5, 3, sumar)); // 8
console.log(operar(5, 3, restar)); // 2

```
En este ejemplo:

1. **operar** es una función de orden superior, porque recibe otra función como argumento **(operacion)**.
2. **sumar** y **restar** son funciones que se pasan como parámetros.
