# `async / await` en JavaScript

## ¿Qué es async?

Una función marcada con **sync** siempre devuelve una Promesa.

Incluso si no se escribe ninguna promesa adentro, JS la envuelve en una automáticamente.

``` javascript

async function ejemplo() {
  return "Hola";
}

ejemplo().then(console.log); // "Hola"

```

## Equivale a:

``` javascript

function ejemplo() {
  return Promise.resolve("Hola");
}

```

## ¿Qué es await?

- Solo se puede usar **dentro de funciones async**.

- Sirve para **esperar el resultado** de una Promesa sin usar `.then()`.

- Mientras espera, **no bloquea el programa** → JS sigue con otras tareas.

``` javascript

async function getData() {
  const response = await fetch("https://restcountries.com/v3.1/name/mexico");
  const data = await response.json();
  console.log(data[0].name.common);
}

getData(); // México

```

## Equivale a usar `.then():`

``` javascript

fetch("https://restcountries.com/v3.1/name/mexico")
  .then(r => r.json())
  .then(d => console.log(d[0].name.common));

```

# `async/await` = azúcar sintáctica

-   `async/await` es azúcar sintáctica sobre Promesas (`resolve / reject`).

-   No añade nada nuevo al motor de JS.

-   Solo hace que el código sea más legible (se parece a código síncrono).
