# Callback
## Mi explicación

`callback` es simplemente una función que se pasa como argumento a otra función, y esa función se ejecuta después de que la otra haya terminado su trabajo.

``` javascript

function saludar(nombre, callback) {
  console.log("Hola " + nombre);
  callback(); // se ejecuta después
}

function despedida() {
  console.log("Adiós!");
}

saludar("Bryan", despedida);

// Salida:
// Hola Bryan
// Adiós!

```

- Aquí, `despedida` es un **callback** que se ejecuta después de `saludar`.

## Callbacks con funciones anónimas

No es obligatorio definir la función aparte, se puede pasarla directamente:

``` javascript

function saludar(nombre, callback) {
  console.log("Hola " + nombre);
  callback(); // se ejecuta después
}

saludar("Bryan", function() {
  console.log("Nos vemos pronto!");
});

// 'Hola Bryan'
// 'Nos vemos pronto!'

```

## Callbacks en operaciones asíncronas

Los **callbacks** son muy usados en tareas que tardan tiempo, como leer archivos, consultar una API, o esperar un temporizador.

``` javascript

console.log("Inicio");

setTimeout(function() {
  console.log("Han pasado 2 segundos");
}, 2000);

console.log("Fin");

// Salida:
// Inicio
// Fin
// Han pasado 2 segundos

```

- Aquí, la función dentro de `setTimeout` es un **callback** que se ejecuta después de 2 segundos.

## Problema de los callbacks → "Callback Hell"

Cuando se encadenan muchos callbacks, el código se vuelve difícil de leer y mantener:

``` javascript

hacerAlgo(function(resultado1) {
  hacerAlgoMas(resultado1, function(resultado2) {
    seguirHaciendo(resultado2, function(resultado3) {
      terminar(resultado3, function() {
        console.log("Todo listo!");
      });
    });
  });
});

```

### Esto se conoce como callback hell o pyramid of doom (pirámide de la perdición).
### Por eso nacieron las Promises y más tarde `async/await`.

## 📌 En resumen:

Un **callback** es una función que se pasa como argumento a otra función para que se ejecute más tarde, ya sea después de una operación sincrónica o asíncrona.
