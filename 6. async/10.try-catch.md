# ¿Qué es `try...catch?`

El bloque `try...catch` sirve para **manejar errores** en el código.

- `try { ... }` → aquí va el código que puede producir un error.

- `catch(error) { ... }` → aquí va el código que se ejecuta si ocurre un error en el bloque `try`.

## Sintaxis básica

``` javascript

try {
  // Código que puede fallar
  let x = JSON.parse("{mal json}");
  console.log(x);
} catch (error) {
  console.log("Ocurrió un error:", error.message);
}

```

- Aquí el `JSON.parse` lanza un error porque el JSON está mal escrito, entonces el `catch` lo atrapa y evita que el programa se rompa.

## Flujo detrás de escena

1. Ejecuta `try {}`
    - Si todo va bien → el `catch` se ignora.

2. Si ocurre un error dentro de `try {}`
    - JavaScript “lanza” ese error `(throw)` y salta inmediatamente al `catch {}`.

3. El `catch(error)` recibe el error como un objeto (`Error`) con propiedades como:

    - `error.message` → el mensaje del error.

    - `error.name` → el tipo de error (ej: `TypeError, SyntaxError`).

    - `error.stack` → la traza de la pila (dónde ocurrió).

# `try...catch` con `async/await`

Cuando trabajamos con `async/await`, necesitamos `try...catch` para manejar los errores porque await puede fallar:

``` javascript

async function getPais(pais) {
  try {
    let res = await fetch(`https://restcountries.com/v3.1/name/${pais}`);
    if (!res.ok) {
      throw new Error(`No se encontró el país. Status: ${res.status}`);
    }
    let data = await res.json();
    console.log(data[0]);
  } catch (error) {
    console.error("Error atrapado:", error.message);
  }
}

getPais("jajaja"); // País inexistente

```

# ✅ En resumen:

- `try` ejecuta código que podría fallar.

- `catch` atrapa y maneja errores (sin romper el programa).

- Con `async/await`, `try...catch` se vuelve esencial para manejar promesas rechazadas.
