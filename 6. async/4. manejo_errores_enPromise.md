## `.catch()`

- Es un método de **Promesas**.

- Se ejecuta solo si la promesa fue rechazada `(rejected)` o si hubo un error en la cadena de `.then()`.

- ermite manejar errores de manera limpia.

```javascript

promesa
  .then(resultado => {
    // si la promesa fue resuelta
  })
  .catch(error => {
    // si la promesa fue rechazada o hubo error
  });

```

## ✅ Ejemplo:

```javascript

fetch("https://restcountries.com/v3.1/name/NoExiste")
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => {
    console.error("Ocurrió un error:", error);
  });

```

- 📌 Si el país no existe o hay un fallo de red → entra al `.catch()`.

## IMPORTANTE 

Intenté este código:

```javascript

const pais = function(pais){fetch(`https://restcountries.com/v3.1/name/${pais}`)
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => {
    console.error("Ocurrió un error:", error);
  })};

  pais("jajaja");

```

Pero mi salió esto:

```javascript

{ message: 'Not Found', status: 404 }

```

Lo que pasa es un detalle de cómo funciona `fetch` en JavaScript:

- 👉 `fetch` **nunca lanza error solo porque la URL devolvió un 404 (Not Found)**.

- Para `fetch`, mientras haya conexión con el servidor, la promesa se resuelve exitosamente.

- El `.catch()` **solo se ejecuta en errores de red (como si no tuvieras internet, DNS caído, etc.)**.

Cuando hago:

```javascript

pais("jajaja");

```

- La API de `restcountries.com` responde con un `status 404`, pero `fetch` lo considera una respuesta válida, por eso no entra al `.catch()`.

👉 Resumen:

### `.catch()` captura errores de red o excepciones que tú lances.

### Si queremos manejar errores de respuesta HTTP (404, 500, etc.), necesitamos usar `response.ok` o `response.status`.

# Ahora con:
# `.finally()`

- También es un método de **Promesas**.

- Se ejecuta siempre, independientemente de si la promesa fue resuelta o rechazada.

- Es útil para tareas de limpieza: cerrar conexiones, ocultar un spinner de carga, resetear valores, etc.

📌 Sintaxis:

```javascript

promesa
  .then(resultado => {
    // si todo va bien
  })
  .catch(error => {
    // si hay un error
  })
  .finally(() => {
    // siempre se ejecuta, éxito o error
  });

```

## ✅ Ejemplo:

```javascript

fetch("https://restcountries.com/v3.1/name/Portugal")
  .then(response => response.json())
  .then(data => console.log("Datos recibidos:", data))
  .catch(error => console.error("Error:", error))
  .finally(() => {
    console.log("Petición finalizada ✅");
  });

```

### Resultado:

```javascript

'Petición finalizada ✅'

```

# Resumen

## Diferencia clave

- `.catch()` → solo maneja errores.

- `.finally()` → siempre se ejecuta, sin importar si hubo éxito o fallo.
