# Promise

Una **Promise** en JavaScript es un objeto que representa el resultado **eventual (futuro)** de una operación **asíncrona**:

- Puede terminar en éxito (resuelta ✅)

- O en error (rechazada ❌).

En lugar de depender de **callbacks anidados**, las **promesas** hacen que el código sea más legible y manejable.

## Estados de una Promise

Una **Promise** puede estar en tres estados:

- **Pending** (pendiente) → cuando aún no se ha completado.

- **Fulfilled** (resuelta) → cuando la operación termina con éxito.

- **Rejected** (rechazada) → cuando ocurre un error.


```javascript

const funcionPromesa = new Promise((resolve, reject) => {
  let exito = true;

  if (exito) {
    resolve("La operación fue exitosa ✅");
  } else {
    reject("Ocurrió un error ❌");
  }
});

funcionPromesa

  .then((resultado) => {
    console.log(resultado); // La operación fue exitosa ✅
  })
  .catch((error) => {
    console.log(error); // Ocurrió un error ❌
  })
  .finally(() => {
    console.log("Promesa finalizada");
  });

```

## Uso con operaciones asíncronas

Las **Promises** son muy útiles cuando algo tarda tiempo, como `setTimeout`, llamadas a APIs, etc.

```javascript

function esperar(ms) {
  return new Promise((resolve) => {
    setTimeout(() => {
      resolve(`Han pasado ${ms / 1000} segundos`);
    }, ms);
  });
}

esperar(2000).then((mensaje) => console.log(mensaje));

// Han pasado 2 segundos

```

## Encadenamiento de Promises (Promise chaining)

Con `then()` se puede encadenar varias acciones sin caer en el **"callback hell"**:

```javascript

function esperar(ms) {
  return new Promise((resolve) => {
    setTimeout(() => {
      resolve(`Han pasado ${ms / 1000} segundos`);
    }, ms);
  });
}

esperar(1000)
  .then((msg) => {
    console.log(msg);
    return esperar(2000);
  })
  .then((msg) => {
    console.log(msg);
    return esperar(3000);
  })
  .then((msg) => {
    console.log(msg);
  });

```

## Ventaja sobre los Callbacks

- Evitan el **callback hell**.

- Manejo de errores más limpio con `.catch()`.

- Se integran perfectamente con `async/await` (que hace el código aún más claro).

## En resumen:
Una **Promise** es una forma más elegante y poderosa de manejar operaciones asíncronas, reemplazando los **callbacks** y haciendo que el código sea más legible, mantenible y fácil de depurar.

# IMPORTANTE

## 1️⃣ fetch(...)

- **Siempre** devuelve una **Promise**.

- Esa promesa, cuando se resuelve, te entrega un objeto **Response** (no JSON todavía).

## 2️⃣ response.json()

- `response.json()` NO devuelve un objeto directamente, devuelve otra **Promise**.

- Esa nueva **promesa**, cuando se resuelve, te da los datos ya en formato JavaScript (lo que antes era JSON en texto, convertido en objeto/array JS).

## 3️⃣ .then() en sí mismo

- El método .`then()` siempre devuelve una nueva **Promesa**.

- Esa promesa se resuelve con lo que retorne la función dentro del `.then()`.

## ✅ En resumen:

- `fetch()` → **Promise** → **Response**.

- `response.json()` → **Promise** → **objeto/array JS** (antes era JSON).

- `.then()` → siempre devuelve otra **Promise**, con lo que retorne dentro.
