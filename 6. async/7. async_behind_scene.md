# La asincronía tras banbalinas

# 1. Execution Context y Call Stack

Cuando se ejecuta código JS, lo hace en un **Execution Context** (contexto de ejecución).

El **Call Stack** (pila de llamadas) es donde se apilan las funciones que se están ejecutando.

## Ejemplo:

```javascript

function a() {
  b();
}
function b() {
  console.log("Hola");
}
a();

```

- `a()` entra al Call Stack → llama a `b()` → `b()` entra al Call Stack → hace el `console.log` → `b()` sale → `a()` sale → stack vacío.

### El Call Stack maneja ejecución síncrona.

# 2. Web APIs (o APIs del entorno)

JavaScript es **single-threaded**, no puede hacer todo solo.
El navegador (o Node.js) proporciona APIs externas para manejar lo asincrónico:

- Timers (``setTimeout``, ``setInterval``).

- Fetch/XHR para llamadas HTTP.

- Event Listeners del DOM.

- APIs de Node.js como ``fs.readFile``.

## Ejemplo:

```javascript

setTimeout(() => console.log("Hola"), 1000);

```

1. `setTimeout` se ejecuta y la Web API del navegador arranca un temporizador.

2. Mientras tanto, el **Call Stack** queda libre para seguir con el resto del código.

3. Cuando el **timer** termina, el **callback** `() => console.log("Hola")` **no entra directo al stack** → va a una cola especial.


# 3. Callback Queue (cola de macrotasks)

Aquí llegan las tareas asíncronas normales después de que las Web APIs terminan:

- ``setTimeout``

- ``setInterval``

- Eventos de DOM

- Algunas tareas de Node.js

### El Event Loop revisa constantemente:

- Si el Call Stack está vacío.

- Entonces toma el primer callback de la **Callback Queue** y lo mete al stack.



# 4. Microtask Queue (cola de microtareas)

Esta es otra cola más prioritaria que la de callbacks.
Aquí van:

- Promises (``.then``, ``.catch``, ``.finally``).

- ``MutationObserver``.

- ``queueMicrotask()``.

### El Event Loop siempre vacía primero la Microtask Queue antes de pasar a la Callback Queue.

```javascript

console.log("Inicio");

setTimeout(() => console.log("Timeout"), 0);

Promise.resolve().then(() => console.log("Promise"));

console.log("Fin");

```

## Orden real de ejecución:

```javascript

Inicio
Fin
Promise   <-- microtask (PROMESA)
Timeout   <-- macrotask (CALLBACK QUEUE)

```

# 5. Event Loop (el coordinador) 🔥

### El Event Loop es el que orquesta todo:

### 1. Ejecuta el código síncrono en el Call Stack.

### 2. Si el stack queda vacío, revisa la Microtask Queue:
    - Saca y ejecuta todas las promesas y microtareas pendientes.

### 3. Luego revisa la Callback Queue:

    - Toma el primer callback y lo mete al stack.

### 4. Repite infinitamente.


## Resumen del flujo asincrónico

- JS ejecuta lo síncrono en el **Call Stack**.

- Lo asíncrono se delega a Web APIs (navegador/Node).

- Cuando termina, el resultado va:

    - A la **Microtask Queue** (promesas, ``queueMicrotask``).

    - O a la **Callback Queue** (timers, eventos, etc.).

- El **Event Loop** mete las tareas de esas colas al **Call Stack** cuando esté libre.

- **Prioridad**: primero microtareas, luego callbacks.

## Ejemplo:

```javascript

console.log("1: Inicio");

setTimeout(() => console.log("2: Timeout"), 0);

Promise.resolve().then(() => console.log("3: Promise"));

console.log("4: Fin");

```

## Flujo:

```javascript

             ┌────────────────────────────┐
             │        Call Stack          │
             └────────────────────────────┘
                       │
                       ▼
          ┌───────────────────────────────┐
          │       Execution Context       │
          │  (donde corre el código sync) │
          └───────────────────────────────┘

1) "Inicio" entra al stack → se ejecuta
2) setTimeout() va a Web API
3) Promise.then() va a Microtask Queue
4) "Fin" entra al stack → se ejecuta

────────────────────────────────────────────
          ┌───────────────┐   ┌─────────────┐
          │   Web APIs    │   │  Event Loop │
          └───────────────┘   └─────────────┘
                │                   │
                ▼                   │
 ┌───────────────────────────┐      │
 │Callback Queue (Macrotasks)│      │
 │   setTimeout, setInterval │      │
 └───────────────────────────┘      │
                                    │
                ▼                   │
 ┌───────────────────────────┐      │
 │  Microtask Queue          │      │
 │  Promise.then, catch, etc.│      │
 └───────────────────────────┘      │
                                    ▼
                              Mueve tareas
                              al Call Stack
────────────────────────────────────────────

```


## Orden de ejecución real

- `console.log("1: Inicio")` → se ejecuta directo.

- ``setTimeout(...)`` → lo maneja **Web API** (se va a la cola de macrotasks).

- ``Promise.then(...)`` → va a la Microtask Queue.

- ``console.log("4: Fin")`` → se ejecuta.

- Event Loop revisa → el **Call Stack** está vacío → ejecuta **Microtask Queue** primero → `3: Promise`.

- Luego ejecuta la **Callback Queue** → `2: Timeout`.

## Salida en consola:

```javascript

1: Inicio
4: Fin
3: Promise
2: Timeout

```
