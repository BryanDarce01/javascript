# forEach

## Con map

``` javascript

const monedas = new Map([
    ['USD', 1],
    ['EUR', 0.85],
    ['JPY', 110]
    ]
);

monedas.forEach(function(valor, clave) {
    console.log(`La moneda ${clave} tiene un valor de ${valor}`);
});

'La moneda USD tiene un valor de 1'
'La moneda EUR tiene un valor de 0.85'
'La moneda JPY tiene un valor de 110'

```

## Con set

``` javascript

const monedas = new Set(['USD', 'EUR', 'GBP'])

monedas.forEach(function(valor, _indice) {
    console.log(valor)
})

'USD'
'EUR'
'GBP'

```

🔍 **¿Qué es ese _indice?**

Aunque parece un "índice", realmente no lo es. En un Set, no hay índices como en un array. Sin embargo, la función forEach aún recibe dos parámetros:

``` javascript

set.forEach((valor, valorRepetido) => { ... })

```

Y en Sets, el segundo argumento es exactamente igual al primero. Esto es simplemente para que la API de ***Set.prototype.forEach()*** sea consistente con ***Array.prototype.forEach()***, que sí recibe **(valor, índice)**.

La convención de usar **_indice** o simplemente _ es una forma de decir: 

"No voy a usar este parámetro".
